写在前面：从 Arch 折腾到 NixOS ​

这个站一开始只是我记命令和记坑点的地方，写着写着就变成了自己的 Linux 生存手册。

这几个月我在发行版之间来回折腾了不少次：先是 Arch，后来换到 CachyOS，又回到 Arch，现在落在 NixOS。中间基本把重装、回滚、换桌面环境、修驱动、调网络、折腾引导这些事情都重新做了一遍。

现在回头看，这些折腾也不算白折腾。很多以前只是“知道个大概”的东西，真的得自己装坏几次、修好几次，才会变成能长期留下来的经验。这个站就是拿来存这些东西的，省得以后又从头踩一遍。

为什么现在在用 NixOS ​

前面几轮折腾之后，我越来越在意几件事：系统配置最好能落到文件里，重装和迁移别太依赖记忆，改坏了能回滚，过一阵子再看时还能说清楚“这台机器为什么会变成现在这样”。

Arch 和 CachyOS 让我学会了很多很底层、也很实用的东西，比如文件系统、引导、图形栈、桌面环境和各种系统服务到底是怎么串起来的。NixOS 对我来说不是“更高级”的系统，更多是把前面这些经验收起来，变成一套更容易复现、也更容易维护的做法。

这个站接下来会写什么 ​

这里不会做成那种面面俱到的大教程，更像长期维护的个人笔记。

大概会一直保留这几类内容：

  • 平时真的会反复遇到的问题，比如校园网、代理、显卡、桌面环境、SSH、Steam
  • 一些值得单独记下来的折腾过程，比如从一个发行版迁到另一个发行版
  • 现在正在慢慢成型的 NixOS / Nix 工作流

所以旧的 Arch / CachyOS 内容我不会全删。它们已经不是我现在的主系统了，但很多经验本身并不过时，尤其是排障、存储、引导和桌面环境这一类东西。

建议怎么读 ​

第一次来看这个站的话，可以按这个顺序：

  1. 先看这篇，知道现在这个站大概想写什么
  2. 再看《Linux 生存手册》，这里面是最杂但也最常用的一批东西
  3. 想看折腾过程，就看《重返 Arch Linux 实录》
  4. 如果你也在用 NixOS，或者准备折腾 Nix，再看《Nix / NixOS 常用命令整理》

近况 ​

接下来一段时间，这个站会慢慢从 Arch / CachyOS 的问题记录，转成更偏 NixOS 的长期维护文档。

但我不打算一下子把旧东西全推倒重写。更现实的做法是：

  • 还能用的内容先留着
  • 太杂的大文再慢慢拆
  • 让每篇文章都更清楚自己在讲什么

如果你也正好在从传统发行版往 NixOS 过渡，希望这里多少能提供一点参考。
